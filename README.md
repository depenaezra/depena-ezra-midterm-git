# Midterm Examination Reflection

## What is Git used for?

Git is a version control system used to track changes in files and manage software development projects. This allows developers to save different versions of their work and to also collaborate efficiently.

## What is the difference between Git and GitHub?

Git is the software used for version control on a local computer, while GitHub is an online platform where Git repositories can be stored, shared, and collaborated on with others.

## Why is branching useful?

Branching allows developers to work on new features or fixes without affecting the main project. This makes collaboration easier as multiple people are able to work independently before merging changes.

## What command did you use to upload your local commits to GitHub?

I used the `git push` command to upload my local commits to my GitHub repository. This command sends my committed changes right from my local repository to the remote repository.

## What did you learn from this activity?

I was able to familiarize how to initialize a Git repository, create commits, work with branches, and merge changes. I am also reminded how GitHub helps store projects online not only for storage but also for collaboration.
